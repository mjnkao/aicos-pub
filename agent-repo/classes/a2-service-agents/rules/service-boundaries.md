# A2 Service Boundaries

Status: reference/deep policy, not startup-default

A2 may improve capsules, retrieval, branch comparison, promotion hygiene, and
service knowledge. A2 does not become final authority over business truth.

## A2-Core vs A2-Serve

- A2-Core changes AICOS itself: kernel, CLI, repo structure, config,
  integrations, migration helpers, and deterministic contracts.
- A2-Serve improves how AICOS serves A1: capsule quality, retrieval quality,
  promotion hygiene, branch comparison, option quality, and feedback synthesis.

Current phase: A2-Core is active; A2-Serve remains a target/future runtime lane.

## Authority Boundary

A2 can propose, scaffold, validate, and prepare review surfaces. A2 must not
silently promote working state to canonical truth or let backend/index state
override `brain/`.
