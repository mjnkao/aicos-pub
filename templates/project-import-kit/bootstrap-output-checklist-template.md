# Bootstrap Output Checklist Template

Status: shared reusable template

Purpose: define minimum outputs before an imported project is startup-usable in
AICOS.

## A. Canonical Minimum

- [ ] `project-brief.md`
- [ ] `product-goals.md`
- [ ] `architecture-baseline.md`
- [ ] `source-limits.md` if needed
- [ ] `data-scope.md` if needed
- [ ] `pipeline-baseline.md` if needed

## B. Working Minimum

- [ ] `working/current-state.md`
- [ ] `working/current-direction.md`
- [ ] `working/handoff/current.md`
- [ ] `working/open-questions.md`
- [ ] `working/open-items.md`
- [ ] `working/active-risks.md`

## C. Evidence Minimum

- [ ] source inventory file
- [ ] import note
- [ ] slice note
- [ ] delivery/reference surface note
- [ ] isolated checkout facts recorded
- [ ] explicit excluded surfaces recorded

## D. A1 Minimum

- [ ] project packet index
- [ ] first review/import packet
- [ ] startup/context validation packet
- [ ] optional continuation packet if needed

## E. Independent Import Test Minimum

- [ ] independent executor can start from the import-kit cluster
- [ ] old-thread memory is not required
- [ ] current working folder is not mutated
- [ ] MCP/runtime implementation remains deferred unless separately approved

## Rule

If bootstrap outputs are not sufficient, the imported project is not yet usable
enough for real A1 work.
