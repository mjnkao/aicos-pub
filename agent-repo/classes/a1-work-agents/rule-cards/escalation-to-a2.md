# Rule Card: Escalation To A2

Load when project work is blocked by AICOS itself.

Escalate to A2 when the problem is:

- AICOS kernel/CLI behavior
- context delivery or capsule quality
- retrieval/sync/indexing
- lane ambiguity
- promotion or rule conflict
- service skill improvement
- architecture change to AICOS

A1 should describe the blocker and options, then stop short of changing AICOS
unless explicitly reassigned as A2.

Do not solve AICOS system friction by adding project-specific A1 rules.
