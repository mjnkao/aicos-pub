# A1 Escalation Rules

Status: reference/deep policy, not startup-default

Escalate when a decision changes architecture authority, destroys old material,
promotes canonical truth, or exposes external cost or privacy risk.

Escalate to A2 when AICOS itself is the blocker: kernel, CLI, context delivery,
retrieval, sync, lane ambiguity, promotion policy, or service skill quality.

Do not encode a project-specific workaround as an A1 core rule when the issue is
really AICOS architecture or service behavior.
