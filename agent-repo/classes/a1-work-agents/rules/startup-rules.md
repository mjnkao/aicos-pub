# A1 Startup Rules

Status: reference/deep policy, not startup-default

A1 agents work on project tasks. Startup is packet-first and project-scoped.

Read the A1 startup card first:

```text
agent-repo/classes/a1-work-agents/startup-cards/a1.md
```

Then read the smallest project orientation set:

- `brain/projects/<project-id>/canonical/project-profile.md`
- `brain/projects/<project-id>/working/current-state.md`
- `brain/projects/<project-id>/working/current-direction.md`

Read `brain/projects/<project-id>/working/handoff/current.md` only for
continuation, interruption recovery, migration/state alignment, or
newest-vs-stale checks.

Future company/workspace rule layers may be added later. Do not load them by
default unless the selected task/project explicitly points to them.

Do not bulk-load raw evidence, all packets, all handoffs, or source design docs
at startup.
