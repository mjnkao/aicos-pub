# Sample Project Authority And Lane Mapping

Status: public sample

## Authority Boundary

- Source repo: code/runtime/scripts/tests/generated artifacts.
- AICOS: compact context/control-plane and continuity.
- Canonical digests: reviewed stable project truth.
- Working files: current active reality only.
- Evidence: source inventory, import notes, and validation notes.

## Canonical Mapping

| source | canonical target | note |
| --- | --- | --- |
| source project README | `canonical/project-brief.md` | digest, do not copy blindly |
| source architecture docs | `canonical/architecture-baseline.md` | active baseline only |
| source data docs | `canonical/source-and-data-scope.md` | source roles and limits |
| delivery docs | `canonical/delivery-baseline.md` | active output surface |

## Working Mapping

| output | purpose |
| --- | --- |
| `working/current-state.md` | current active reality |
| `working/current-direction.md` | current direction |
| `working/open-items.md` | active queue |
| `working/open-questions.md` | unresolved questions |
| `working/active-risks.md` | active risks |
| `working/handoff/current.md` | current continuation surface |

## Rule

Normalize into compact AICOS lanes. Do not mirror the source repo tree.
