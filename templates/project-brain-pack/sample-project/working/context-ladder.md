# Sample Project Context Ladder

Status: template
Scope: `projects/sample-project`

## Purpose

Give human managers and AI agents the smallest useful read path before doing
project work.

## Layer 0 - Project Identity

Read:

- `brain/projects/sample-project/canonical/project-profile.md`

Learn:

- project scope
- authority boundary
- actor expectations

## Layer 1 - Current Working Reality

Read:

- `brain/projects/sample-project/working/current-state.md`
- `brain/projects/sample-project/working/current-direction.md`

Learn:

- current status
- active direction
- non-goals

## Layer 2 - Continuation

Read only when continuing existing work:

- `brain/projects/sample-project/working/handoff/current.md`

Learn:

- latest handoff summary
- recent ownership and next safe action

## Layer 3 - Active Work

Read after orientation:

- `brain/projects/sample-project/working/open-items.md`
- `brain/projects/sample-project/working/status-items/`

Learn:

- open work
- blockers
- follow-up items

## Layer 4 - Evidence

Read only when a task points to it:

- `brain/projects/sample-project/evidence/`

Learn:

- source inputs
- supporting research
- audit/provenance records

## Not Read By Default

- all evidence files
- old handoff episodes
- unrelated project brains
- generated runtime/cache files
